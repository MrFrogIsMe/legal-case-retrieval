# 資料設計 v1：子集篩選策略與排除規則

> 對應腳本：`experiments/03_build_corpus.py`
> 本文件記錄「哪些案件被納入、哪些被排除、為什麼」，作為未來可重現性與透明度的依據。

---

## 1. 資料來源與專案收斂

- **來源：** 司法院開放資料（opendata.judicial.gov.tw）
- **專案收斂決策：** 
  本專案最終決策**收斂至「僅針對刑事案件」進行分析**。
  - *原因 1 — 語意聚焦：* 刑事案件（車禍過失傷害、酒駕、肇逃）具有極度固定的「事實段」自然語言描述，非常適合 RAG 的語意相似度檢索（Dense Retrieval）。
  - *原因 2 — 避免混亂：* 民事判決主要著重於繁雜的「金額計算（折舊、看護費等）」與「過失相抵責任比例」，混入民事會使檢索語意過於雜亂，難以在有限的 Prompt 內兼顧民刑兩大體系。
- **原始量級：** 刑事全量共 532,276 筆（本專案完全排除 1,074,639 筆民事案件）。

---

## 2. 時間與年份篩選（近 10 年）

只納入**近 10 年（民國 105 年至 114 年，西元 2016-2025 年）**的刑事判決。

**理由：**
- 控制資料集量級於合理實驗規模。
- 近 10 年的量刑、社會見解與法律見解，最具備現實參考與警示價值。
- 排除 104 年以前太舊的判決（刑事全量中約有 **368,136 筆** 因太舊而被排除）。

---

## 3. 審級與法院限制：只做地方法院（一審）

本專案**完全排除高等法院（二審）與最高法院（三審）之判決，僅保留地方法院一審判決**。

**決策原因：**
1. **職責不同（Facts-rich vs Law-heavy）：**
   - **地方法院（一審 — 事實審）：** 法官必須像偵探一樣說故事，在判決書「事實段」中詳細記載人、事、時、地、物（如「時速多少、未注意車前狀況、撞倒某乙骨折」）。這種富含事實的自然語言語料（Facts-rich），是 RAG Embedding 相似度匹配的靈魂。
   - **高等/最高法院（二/三審）：** 二審通常沿用一審事實、著重於量刑爭辯；三審為「純法律審」，只討論法律程序，判決書內容極度抽象且充斥文言法律爭點。這類語料缺乏事實故事，AI 拿去比對根本無法對齊使用者的口語事由。
2. **避免「一案三吃」的檢索冗餘（Retrieval Redundancy）：**
   若不限制審級，同一宗車禍案件上訴到最高法院會產生 3 筆判決（一審 + 二審 + 三審）。這會導致 AI 在檢索時，可能重複吐出同一個案件的 3 種審級版本，佔滿了 Top-5 的檢索版面，嚴重降低回答的多樣性。
3. **最貼近使用者切身焦慮：**
   一般人若涉入車禍或酒駕刑事糾紛，最焦慮的是「我如果被告，法庭（第一線地方法院一審）通常會怎麼判我、會判幾個月、能不能易科罰金」。地院一審判決最符合使用者的切身需求。

---

## 3. 第一刀：排除程序性案件

程序性案件**沒有事實描述與法律推理**（如「支付命令」只有一行「命被告給付 XX 元」），對 RAG 類案檢索完全沒有意義，應在最前端排除。

## 4. 第一刀：排除刑事程序性案件

刑事程序性案件**沒有事實描述與法律推理**（例如「定應執行刑」只是單純把多個判決刑期併成一個執行刑，不含任何車禍或酒駕的犯罪情境描述），這類案件對 RAG 類案檢索完全沒有意義，應在最前端排除。

### 4.1 刑事排除規則

| 排除條件 | 具體判斷 | 實際過濾量 |
|---------|---------|------------|
| JCASE 含「聲」 | 聲明異議、各種聲請程序事件 | ~13,503 筆 |
| 案由含「聲明異議」 | 同上（部分案由以文字標示） | 含於上 |
| 案由為「定應執行刑」 | 純量刑程序，無事實 | 少量 |
| 案由為「宣告沒收」 | 純沒收程序 | 少量 |

**刑事排除邏輯（程式碼）：**
```python
def is_criminal_procedural(jcase: str, title: str) -> bool:
    if "聲" in jcase: return True
    if title in ("定應執行刑", "宣告沒收", "聲明異議"): return True
    return False
```

---

## 5. 第二刀：分層抽樣（控制量級，保留多樣性）

### 5.1 為什麼要分層？

排除程序性與非地院判決後，常見案由如「公共危險（酒駕）」有高達 19,791 筆，而「過失致死」只有 370 筆。如果直接隨機抽樣，大類（酒駕）會完全壓過小類（重大車禍致死），導致系統對少見或嚴重案件類型的檢索與回答能力極差。

### 5.2 分層策略

- **分層依據：** `JTITLE`（案由）
- **每層上限：** **50 筆**（`PER_TITLE_CAP = 50`）
- **邏輯：** 若該案由 ≤ 50 筆 ➔ 全部保留；若 > 50 筆 ➔ 隨機抽出 50 筆（隨機種子 seed=42）。

## 7. 不納入說明

| 類別 | 排除原因 |
|------|------|
| **民事判決（全部）** | 民事訴訟著重折舊、求償、過失比例，與刑事之罪名、有罪與否之體系截然不同。避免 RAG 語意檢索雜亂，本專案完全排除民事。 |
| **高等/最高法院（全部）** | 二審沿用事實著重重判、三審純法律審內容極度抽象，兩者均缺乏詳細的自然語言「故事事實描述」；且會引入「一案多吃」的檢索冗餘（Retrieval Redundancy）。 |
| **行政判決 / 懲戒法院** | 非本專案車禍與民生刑事探討之範疇。 |
| **民國 104 年前判決** | 量刑、緩刑標準、社會觀感與當今司法實務有通膨/通期落差，警示參考價值低。 |
