# 實驗 02：結構切段 — 結果筆記

腳本：`experiments/02_segment.py`
核心邏輯：`src/lcr/data/segment.py`
探查腳本：`experiments/probe_segments.py`（設計 regex 前先實測標記模式）

## 關鍵發現：判決書有兩種結構

實測（probe_segments.py，2000 份抽樣）發現分段標記因案件類型而異：

- **簡易判決（交簡類）**：主文 + **事實及理由**（合併段，不分開），佔比最高
- **通常判決（交易/交訴類）**：主文 + 事實(或犯罪事實) + 理由
- 主文 95.3% 出現，是最穩定的錨點
- 「交附民」（附帶民事）多無任何標記，結構特殊

→ 切段不能只找「事實」，否則漏掉近半的簡易判決。
  `segment.py` 以主文為錨，中段支援 事實/犯罪事實/事實及理由，並處理
  「事實及理由」與「事實」的重疊命中（保留較長者）。

## 執行結果

| 指標 | 數值 |
|------|------|
| 子集輸入 | 59,819 |
| 排除聲明異議（JCASE 含『聲』或案由含『聲明異議』） | 10,466 |
| 讀檔失敗 | 0 |
| **切段輸出** | **49,353** |
| **完整率（主文 + 事實或理由）** | **85.3%**（對齊 design 目標 >85%） |

### layout 分布

| layout | 數量 | 比例 | 說明 |
|--------|------|------|------|
| merged | 23,555 | 47.7% | 簡易判決，事實及理由合併段 |
| partial | 14,744 | 29.9% | 缺事實或缺理由（細分見下） |
| three_part | 8,496 | 17.2% | 標準三段 |
| none | 2,558 | 5.2% | 無標記，fallback 全文塞 reasoning |

### partial 細分（重要，影響 embedding 選段）

| 組成 | 數量 | 處置 |
|------|------|------|
| main + 理由（無事實） | 8,016 | 理由段含事實敘述，**可用** |
| 只有 main | 4,669 | 品質差，下游可考慮排除或特別標記 |
| main + 事實（無理由） | 2,058 | 有事實可用 |

## 對下游（embedding）的啟示

**選段策略：embedding 不能只認 facts 段，須 fallback。**
建議優先序：`facts → reasoning → main`。
因為 8,016 筆「主文+理由」結構的理由段其實包含事實敘述；
而 merged 的合併段同時填入 facts 與 reasoning，任一皆可。
「只有 main」的 4,669 筆（約 9.5%）內容最少，可在 embedding 階段評估是否排除。
