## 執行結果

### Phase 1：遍歷 + 排除

| 指標 | 數值 |
|------|------|
| 總掃描（刑事+民事） | 1,606,915 |
| 跳過（非地方法院） | 111,730 |
| 跳過（年份 < 民國100年） | 721,865 |
| 跳過（程序性案件） | **400,876** |
| 讀檔失敗 | 1 |
| **Phase 1 候選** | **372,443（6,907 種案由）** |

程序性排除量（40 萬筆）比預期更多，代表原始資料確實充斥大量無法律論述的程序性裁定。

### Phase 2：分層抽樣（200 筆/案由）

| 指標 | 數值 |
|------|------|
| **最終語料庫** | **85,131 筆** |
| 刑事 | 22,243 |
| 民事 | 62,888 |
| 案由種類 | 6,907 種 |
| 截頂（>200筆，抽樣） | 179 種 |
| 完整保留（≤200筆） | 6,728 種 |

絕大多數案由（6,728/6,907 = 97.4%）筆數本就 ≤ 200，全部保留，
多樣性良好，分層抽樣沒有過度截斷。

## 關鍵觀察

1. **民事量是刑事的 2.8 倍**（62,888 vs 22,243）：
   因民事原始量更大，即使抽樣後仍較多。
   若後續 embedding 需平衡比例，可考慮對民事再設全局上限（如 3 萬筆）。

2. **6,907 種案由多樣性高**：
   系統能回答各種案件（不只車禍），涵蓋刑事（公共危險/詐欺/傷害）
   與民事（損害賠償/房屋/分割/執行）等廣泛主題。

3. **85,131 筆對 embedding 偏多**：
   BGE-M3 embedding 全量跑約需 2-4 小時（依機器），
   若要加快實驗迭代速度，可對整體再隨機抽 3-5 萬筆做初步實驗，
   確認 pipeline 正確後再全量跑。
